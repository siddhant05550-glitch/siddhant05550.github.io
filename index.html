// 001 · Featured Project In Progress
Smartphone Sensor Usage — ML Classification System
An end-to-end machine learning pipeline that analyses smartphone sensor data to classify high-usage or overheating conditions. The system processes real sensor features — screen time, motion sensor readings, touch event frequency, battery consumption, and device activity levels — through a complete ML workflow: raw data ingestion → preprocessing → EDA → feature selection → model training → evaluation.
Python scikit-learn Pandas NumPy Matplotlib Seaborn PCA EDA Classification
GitHub View Project
Full Project Pipeline
  • Smartphone sensor dataset selection — screen time, motion, touch events, battery, activity levels
  • Data preprocessing: cleaning, missing value handling, normalisation, feature transformation
  • Exploratory Data Analysis — feature distributions, usage patterns, sensor-battery correlations, outlier detection via statistical summaries and visualisations
  • Correlation analysis to identify dominant heavy-usage indicators
  • Feature selection using PCA, Forward Selection, and Backward Selection based on EDA insights
  • Classification model development: normal vs heavy usage and overheating scenario detection
  • Full model evaluation — accuracy, precision, recall, F1-score, and confusion matrix analysis
